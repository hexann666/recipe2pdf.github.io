Scrum setup

To structure the tasks in the project I devided all Issues into 7 Steps, 6 of them are provided as required content in the project requirements and Step 0 is dedicated to SCRUM workflow:

Step 0 - prepare canvas for SCRUM

Step 1 - requirements based on the project goal

Step 2 - describe Data Science methodology concept

Step 3 - diagrams

Step 4 - web technologies

Step 5 - architecture (optional)

Step 6 - desing and DS architecture

To follow the Issues assigned to each Step, there is an Epic board, that shows all Issues according to the Epics they belong to.

To track the progress of the Issues there is the Kanban board structured after defined Issue states:

  • open - automatically given to any new Issue by GitLab if nothing else specified;
  • in progress - work has begun;
  • blocked - work is blocked by another not-done Issue;
  • done - work is finished.
